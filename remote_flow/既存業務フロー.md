# 既存業務フロー

## 在宅勤務申請

- 課題
  - 定型のメールを毎回作成する無駄(5分→1分×室員の改善)
  - 上長のメール承認の煩雑さ(1分→30秒×室員の改善　※室員分だけ承認するため)
  - 上長の承認履歴の確認の煩雑さ(2分→30秒×室員の改善　※室員分だけ承認するため)

##### 既存業務フロー

```mermaid
sequenceDiagram
    participant Staff as 申請者
    participant Boss as 上長
    Note over Staff, Boss: 在宅勤務申請（メール）

    Staff->>Boss: 申請メールを送付
    Boss-->>Boss: 内容の確認・検討
    Boss->>Staff: 承認（または却下）の返信
    Note right of Staff: 申請完了
```

## 在宅/出張報告

- 課題
  - 定型チャットを毎回作成する無駄(3分→1分×室員の改善)
  - 報告忘れ
  - Teamsのチャットが埋もれ、報告履歴の確認の煩雑さ

```mermaid
sequenceDiagram
    participant Staff as 申請者
    participant Boss as 上長

    Note over Staff, Boss: 在宅/出張開始報告（Teams）
    Staff->>Boss: Teamsチャットを送付
    Boss-->>Boss: 内容の確認
    Staff-->>Staff: 業務

    Note over Staff, Boss: 中抜け報告（Teams）
    Staff->>Boss: Teamsチャットを送付
    Boss-->>Boss: 内容の確認
    Staff-->>Staff: 業務

    Note over Staff, Boss: 在宅/出張終了報告（Teams）
    Staff->>Boss: Teamsチャットを送付
    Boss-->>Boss: 内容の確認
```

## 備品貸出

- 課題
  - 貸出機でしか備品の貸出状況が把握できていない
- 効率化
  - 2 備品保管場所→貸出機(30秒→0秒)
  - 4 ユーザー読み込み(10秒→0秒)
  - ★現品確認を容易にしたい

```mermaid
graph TD
    Start((開始)) --> Search[1. 備品を探す]
    Search --> Move[2. 貸出機まで移動]
    Move --> ScanItem[3. 備品の読み込み<br/>QR]
    ScanItem --> ScanUser[4. ユーザー読み込み<br/>社員証]
    ScanUser --> Process[5. 貸出手続き<br/>システム登録]
    Process --> Complete((完了))

    style Start fill:#f9f,stroke:#333,stroke-width:2px
    style Complete fill:#f9f,stroke:#333,stroke-width:2px
```
