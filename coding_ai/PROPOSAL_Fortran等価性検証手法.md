# Fortran 数値解析コードの移行等価性検証のための物理法則に基づくテスト仕様自動生成

作成日: 2026-06-24

---

## 1. 研究背景

数値解析分野では Fortran で実装されたプログラムの Python 等への移行が求められている。保守人材の減少とハードウェア環境（GPU/ヘテロジニアス計算）への非対応が主因であり、移行を LLM で支援する研究が行われている \[1\]\[2\]。

しかし LLM による直接翻訳では、翻訳の最大 77.8% がコンパイル不能か誤結果を生む \[3\]。振る舞い等価率（BER）は最良構成でも 9.4〜19.4% にとどまり \[4\]、構文的に正しい翻訳が振る舞い等価性をサイレントに破壊する事例が報告されている \[4\]。移行の合否は等価性の検証にかかるが、数値解析プログラムの等価性検証には構造的な困難がある。

---

## 2. 課題

### 2.1 オラクル問題

数値解析プログラムでは「正解出力」が存在しないまたは定義できないことが多い。これを**オラクル問題**と呼び、科学計算ソフトウェアのテストにおける中核課題として確立されている \[5\]。

この問題への対応として、移行前のレガシー出力を正解（オラクル）として使う方法がある。Fortran→C++ の評価では出力をトークン化・float 変換し許容差付きで比較する手法が実証されている \[6\]。しかしこの方法はレガシー固有の数値誤差を「正解」として固定するリスクを持つ。レガシー出力に依存しないオラクルが必要である。

### 2.2 物理法則は言語・実装に依存しないオラクルである

数値解析プログラムが対象とする物理現象は、実装言語に関わらず保存則・対称性・スケール則・収束次数などの不変量を満たす。これらをオラクルとすれば、レガシー出力に依存せず移行コードの正しさを検証できる。

この考え方はメタモルフィックテスト（MT）として確立されており \[5\]、1998 年以降の MT 応用の 4 分の 1 超が科学計算ソフト（PDE・モンテカルロ等）である。海洋シミュレーションでは既知の物理的対称性をメタモルフィック関係（MR）として定義し、同一モデルの 2 実装を突き合わせる等価性検証が実証されている \[7\]。この構造は移行前後の 2 実装を比較する本課題と本質的に同一である。

### 2.3 MR の人手定義がボトルネックである

MT の適用には MR の定義が必要だが、これは従来ドメイン専門家による人手作業であった。数値解析プログラムの対象ドメインは多岐にわたり、ドメインごとに専門知識が必要なため、MR の定義が MT 適用のボトルネックとなっている \[8\]。

LLM によるプログラム修復（APR）タスクへの MT 適用では、LLM が意味的等価な変換に基づく MR をトークン・文・ブロックの 3 粒度で 9 種類自動生成し、平均 34.4〜48.5% のテストで不安定性を検出した事例がある \[9\]。ただし対象は一般的なプログラムであり、物理法則を根拠とした MR の LLM 生成は未確立である。

## 3. 提案手法

**LLM が物理法則に基づく MR を自動生成し、人間が承認してから移行コードの等価性を検証する**手法を提案する。

### 3.1 手法の構成

**フェーズ 1 ― LLM による MR 生成**

移行元の Fortran ソースコードを LLM に入力し、そのプログラムが対象とする物理現象に対して成立する不変量を MR として生成させる。生成する MR の類型は次の 3 種とする。

- **対称性 MR:** 入力の対称的変換（符号反転・座標変換等）に対して出力が予測可能に変化する関係。海洋モデルの移行検証で実証済みの構造を参照する \[7\]。
- **スケール則 MR:** 入力をスケーリングしたときに出力が物理法則に従って変化する関係（例: 線形弾性では荷重 2 倍で変位 2 倍）。
- **保存則 MR:** 閉じた系でのエネルギー・質量・運動量の保存が成立する関係。

これら 3 類型は科学計算ソフトのテストにおいて物理的不変量を MR として使用する既存研究に基づく \[5\]\[7\]\[10\]。

**フェーズ 2 ― 人間による MR の承認**

LLM が生成した MR を人間（ドメイン専門家）が点検・承認する。承認されなかった MR は廃棄するか修正する。承認済み MR のみを等価性検証に使用する。

この承認ゲートは AgentModernize の Behavioral Specification Graph（BSG）が採用した glass-box 構成に対応する。BSG では「LLM が理解した内容」と「生成コード」の間に人間が点検できる信頼境界を置き、仕様からテストオラクルを生成する構成として実証されている \[4\]。本提案はその構成を MR 生成に適用する。

**フェーズ 3 ― MR による等価性検証**

承認済み MR を移行前後の両コードに適用する。MR に反するケースを等価性違反の候補として記録する。

### 3.2 手法の限界

- MR はプログラムが持つ物理的不変量の一部を捉えるものであり、全ての等価性違反を検出する保証はない \[7\]。
- LLM が生成した MR に誤りが含まれる可能性があるため、人間の承認（フェーズ 2）が不可欠である。
- 物理的不変量を持たない業務ロジック部分の等価性は本手法の対象外である。

## 4. 評価計画

技術的課題は 2 つある。(A) オラクル問題：レガシー出力を正解とすると、レガシー固有の誤差を「正解」として固定する。(B) MR 定義のボトルネック：従来、物理法則に基づく MR の定義は人手による専門家作業であった。評価はこれら各課題が本手法によって解消されているかを問う。

### 4.1 評価対象

オープンソースで公開されている Fortran 数値解析プログラムのうち、移行前後の両コードが実行可能な状態で取得できるものを使用する。

課題 (A) の評価に必要な ground truth として、**出力比較では通過するが物理的不変量を破るミューテーション**を用意する。具体的には、元コードの数値スキームに対して「数値出力はテスト入力の範囲内で許容差内に収まるが、対称性・保存則・スケール則の少なくとも 1 つを意図的に違反させる」変更（係数の微調整・数値積分の次数低下・境界条件の微変更等）を注入する。これにより「出力比較が見逃すが MR が検出すべき等価性違反」を制御された条件で生成できる。

### 4.2 研究課題と評価指標

**RQ1（課題 A への対応）: 出力比較が見逃す物理的等価性違反を MR で検出できるか**

出力比較で通過したミューテーションのうち、承認済み MR が違反と判定した割合を測定する。この割合が有意であれば、物理法則に基づく MR がレガシー出力オラクルの限界を補完することを示す。

| 指標 | 定義 |
|---|---|
| 出力比較通過・MR 検出率 | 出力比較（許容差付き）を通過したミューテーションのうち承認済み MR が違反を検出した割合 |
| 出力比較検出・MR 通過率 | 出力比較が違反と判定したケースのうち MR が通過させた割合（両手法の補完関係の確認） |

**RQ2（課題 B への対応）: LLM が生成した MR は人手作成の MR を代替できるか**

同一プログラムに対して (i) ドメイン専門家が人手で作成した MR と (ii) LLM が生成し人間が承認した MR を用意し、両者をミューテーション検出率で比較する。LLM 生成 MR が人手 MR と同等の検出率を達成すれば、MR 定義のボトルネックが解消されることを示す。

| 指標 | 定義 |
|---|---|
| MR 一致率 | LLM 生成 MR と人手 MR が同一の物理的不変量を捉えている割合 |
| 検出率差 | 人手 MR の検出率と LLM 生成 MR（承認後）の検出率の差 |
| MR 修正率 | 承認フェーズで人間が修正を加えた MR の割合（修正工数の指標） |

### 4.3 評価環境の統制

コンパイルフラグ欠落・ライブラリ未リンク等の評価環境の不備は偽の失敗を生むことが 6,164 翻訳の分析で確認されている \[11\]。構成情報を明示した再現可能な評価環境を用意し、環境由来の失敗と等価性違反を区別する。
